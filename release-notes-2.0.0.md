# CPAN::Maker::Bootstrapper 2.0.0 Release Notes

## Overview

Version 2.0.0 is a major release that migrates the Bootstrapper to a
role-based architecture using `CLI::Simple` 2.0.0, adds a
comprehensive annotation workflow with comments and `.annotate` file
support, introduces `CLI::Simple` scaffold import capability, and
includes significant build system improvements.

## New Architecture

**Role-based implementation via CLI::Simple 2.0.0.** The Bootstrapper
is now itself a role-based `CLI::Simple` application, driven by
`cpan-maker-bootstrapper.yml`. Commands are implemented in dedicated
`Role::Tiny` roles:

- `Role::Installer` - `install` / `default`
- `Role::CreateConfig` - `create-config`
- `Role::Init` - startup initialization
- `Role::ScaffoldImporter` - `import-scaffold`
- `Role::LLM::Annotator` - `annotate`, `update-annotations`
- `Role::LLM::Reviewer` - `code-review`, `pod-review`, `code-finding`, `pod-finding`
- `Role::LLM::Models` - `models`
- `Role::LLM::ReleaseNotes` - `release-notes`
- `Role::LLM::Utils` - shared LLM utilities

## New Features

**`import-scaffold` command.** Consumes a tarball generated by
`cli-simple -scaffold` and installs it as a new Bootstrapper project
in one step:

```bash
bootstrapper import-scaffold my-script-roles.tar.gz \
  --module My::Script --installdir .
```

The tarball is extracted to a temporary directory and fed directly to
`cmd_install` via the `--import` path.

**Annotation comments.** The `-a` option now supports inline comments
via a colon-delimited third field:

```bash
bootstrapper annotate -a 1:ACCEPT:"fixed in 2.0.0"
```

A `--comment` option applies a shared comment to all findings in a
`-a` batch:

```bash
bootstrapper annotate --comment "fixed in 2.0.0" -a 1:ACCEPT 3:ACCEPT 6:ACCEPT
```

Inline comment takes precedence over `--comment`. Existing comments
are preserved when no new comment is supplied.

**`.annotate` file workflow.** A structured INI-format file for bulk
annotation without the command line:

```bash
bootstrapper annotate --output lib/CPAN/Maker/Bootstrapper.pm.in
# edit the generated .annotate file
bootstrapper update-annotations lib/CPAN/Maker/Bootstrapper.pm.in
```

The `update-annotations` command generates the `.annotate` file if it
doesn't exist, or applies it to the review if it does. The `-a`
command keeps the `.annotate` file in sync after each annotation.

**Model inheritance across review cycles.** The model used for a
review is saved in the review JSON and inherited by subsequent reviews
of the same file. `--model` overrides the inherited model and resets
the lineage, with a warning when the model changes mid-cycle.

**Pre-submission cost estimates.** Before submitting to the LLM API, a
token and cost estimate table is displayed showing prompt, file,
annotation, and context token counts with per-component cost
breakdowns.

**`_show_models` summary table.** The `models` command now displays a
formatted `Text::ASCIITable` summary including model ID, display name,
context window, output limit, release date, and capabilities.

**`modulino.tmpl` now sets `MODULINO_WRAPPER`.** Generated bash
wrappers correctly set `MODULINO_WRAPPER` for `CLI::Simple` 2.0.0 bash
completion support.

## Build System Improvements

**`perl.mk` improvements:**
- `tidy` target now only builds modules and scripts rather than the
  full distribution, avoiding unnecessary tarball rebuilds
- Syntax check errors are now captured and displayed rather than
  silently discarded
- `CLEANFILES` uses `find-files` macro to correctly locate `.tdy`,
  `.crit`, and `.ERR` files in subdirectories
- `@` suppression replaced with `$(NO_ECHO)` throughout for
  consistency
- Perltidy and Perlcritic output suppressed during lint checks - only
  errors are shown

**`git.mk` improvements:**
- `make git` now runs `make clean` first
- Initial `ChangeLog` entry is auto-generated with a file listing at
  project creation time
- `README.md` added to recommended artifacts

**`project.mk`** - now uses `find-files` macro to discover role files
automatically, so new roles are picked up without manual `project.mk`
edits.

**`help.mk`** - `ALIAS` variable documented for modulino wrapper
naming.

## Dependency Updates

- `CLI::Simple` bumped to 2.0.0
- `CPAN::Maker` bumped to 1.8.0
- `File::Copy::Recursive` added
- `Role::Tiny` and `Role::Tiny::With` added
- `+` prefix syntax in `requires` used for documentation-only
  dependencies

## Bug Fixes

- `File::Copy::Recursive::dircopy` replaces `rename` in
  `cmd_install` - correctly merges into existing installation
  directories including `--installdir .`
- `.includes/*.mk` files no longer marked read-only, allowing `make
  update` to overwrite them
- `JSON::PP->new->decode` (without `->utf8`) used for decoding LLM
  JSON responses - fixes "Wide character" errors on re-review passes
- `File::Find` replaces `glob` in `_get_latest_review` -q eliminates
  glob metacharacter sensitivity in filenames
